# Regras de Desenvolvimento do Projeto

- **Controle de Versão (Git)**:
  - Faça apenas commits locais (`git commit`) para registrar o progresso das tarefas.
  - **NÃO** envie as alterações para o repositório remoto do GitHub (`git push`) de forma proativa.
  - A operação de `git push` para o GitHub só deve ser executada após uma solicitação explícita do usuário.
